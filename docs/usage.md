# Использование

[English version](usage.en.md)

## Mini App

Главная страница показывает серверы, доступность панели и x-ui, ОС и число клиентов. Страница сервера содержит системные метрики, управление пользователями, сведения о версии и кнопки обновления.

### Пользователи

1. Откройте сервер.
2. Нажмите «Пользователи».
3. Выберите inbound.
4. Укажите имя, срок, трафик и лимит устройств.
5. После создания откроется ссылка и QR.

Если один клиент привязан к нескольким inbound, диалог предложит выбрать конфигурацию. Ссылки поступают из API 3x-ui, поэтому учитывают Reality, XHTTP, Hysteria2, external proxy и другие настройки панели.

### Обновление ОС

Кнопка видна только при настроенном `system_update_command`. После подтверждения backend создаёт фоновое задание, запускает wrapper по SSH и сохраняет результат в `data/jobs`.

Wrapper выполняет `apt-get update` и `apt-get upgrade --with-new-pkgs`. Reboot автоматически не производится.

### Обновление 3x-ui

Кнопка появляется, когда GitHub сообщает о более новой стабильной версии и настроен `panel_update_command`. Перед запуском база сохраняется через API на управляющем сервере и локально на управляемом сервере.

## Команды Telegram

```text
/servers
```

Список ID серверов.

```text
/status de-1
```

Сервис, версия, uptime, load и диск.

```text
/logs de-1 100
```

Последние 10–500 строк журнала x-ui.

```text
/updates de-1
```

Пакеты, доступные по текущему apt cache. Команда не выполняет `apt-get update`.

```text
/ssh de-1 uname -a
```

Работает только при `enable_raw_ssh=true`. Команда выполняется от `xuiadmin`, без PTY, с timeout и ограничением вывода. Не отправляйте через Telegram пароли, токены или ключи.

## Аудит

Изменяющие операции записываются в `data/audit.jsonl`. Raw SSH сохраняет hash команды и exit status, а не её текст.

## Резервные копии

API-копии базы находятся в `data/backups`. Срок хранения задаётся `BACKUP_RETENTION_DAYS`. Локальные копии управляемого сервера находятся в `/var/backups/x-ui`.
