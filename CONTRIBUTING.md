# Участие в разработке

1. Создайте issue с описанием задачи или ошибки.
2. Создайте отдельную ветку.
3. Не добавляйте секреты и реальные клиентские ссылки.
4. Добавьте тест на изменяемое поведение.
5. Выполните `make validate`, `make test` и `make syntax`.
6. В pull request опишите совместимость с версией 3x-ui и способ ручной проверки.

Для изменений API панели используйте обезличенные fixtures. UUID, пароли клиентов, Reality keys, Telegram tokens, IP-адреса реальных серверов и содержимое `x-ui.db` запрещены в issues и commits.

Изменения update wrappers требуют особенно внимательного review: они выполняются через sudo на управляемых серверах.
